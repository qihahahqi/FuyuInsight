## 报告安全漏洞

如果您发现安全漏洞，请不要通过公开的 GitHub Issue 报告。请通过以下方式私下联系我们：

- 发送邮件至：[在此处填写安全联系邮箱]

请在报告中包含以下信息：

1. 漏洞类型（如 XSS、SQL注入、认证绕过等）
2. 受影响的功能路径和参数
3. 复现步骤
4. 可能的影响范围
5. 如果有，提供修复建议

我们承诺：

- 在 48 小时内确认收到您的报告
- 在 7 天内初步评估漏洞严重程度
- 及时通知您漏洞处理进度
- 在修复后公开致谢（如果您愿意）

## 安全最佳实践

使用本系统时，请注意以下安全事项：

### 配置安全

- `config/config.yaml` 包含敏感信息，请勿提交到版本控制
- API Key 应使用环境变量或安全存储
- 数据库密码应定期更换

### 部署安全

- 生产环境应使用 HTTPS
- 设置适当的防火墙规则
- 定期备份数据库
- 限制 API 访问频率

### 用户安全

- 不要在公开场合分享您的配置文件
- 定期检查账户活动
- 使用强密码保护数据库

## 已知安全措施

本系统已实施以下安全措施：

- 用户认证和会话管理
- SQL 注入防护（使用 SQLAlchemy ORM）
- XSS 防护（前端数据转义）
- CSRF 保护（待实现）
- API 访问控制（登录验证）
