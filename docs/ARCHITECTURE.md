### 4.1 典型请求流程

```

   │                │                  │                  │
   │── 点击"持仓"──▶│                  │                  │

   │                │                  │── 查询 positions─▶│

```

## 6. 技术选型说明

| 层级 | 技术 | 选择理由 |
|-----|------|---------|
| 前端 | 原生 JS | 轻量、无依赖、快速开发 |
| 图表 | Chart.js | 成熟、美观、易用 |
| 后端 | Flask | 灵活、插件丰富、适合中小项目 |
| ORM | SQLAlchemy | 功能强大、防 SQL 注入 |
| 数据库 | MySQL | 成熟、稳定、开源 |
| 异步 | threading | 个人项目足够，Celery 过重 |
| AI | OpenAI/Anthropic | 效果好、API 标准 |
